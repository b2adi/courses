## 📝 Sample Interview Questions & Answers

### Question 1: "Explain closures"
**Better Answer:**
"A closure is a function that has access to variables from its outer scope even after the outer function has returned. This happens because functions in JavaScript form closures around the data they need. Closures are useful for data privacy - we can create private variables that can only be accessed through specific functions. For example, I could create a counter where the count variable is private and only accessible through increment and decrement methods."

### Question 2: "What's the difference between var, let, and const?"
**Better Answer:**
"All three declare variables, but with important differences. Var is function-scoped and hoisted, meaning it can cause bugs due to hoisting. Let and const are block-scoped and have a temporal dead zone, preventing access before declaration. Const prevents reassignment but not mutation of objects. In modern JavaScript, I'd use const by default, let when I need to reassign, and avoid var completely."

### Question 3: "Explain the event loop"
**Better Answer:**
"JavaScript is single-threaded but can handle asynchronous operations through the event loop. When you execute code, the call stack runs synchronous code. For asynchronous operations, callbacks are placed in either the microtask queue (Promises) or macrotask queue (setTimeout). After the call stack is empty, the event loop checks the microtask queue first, then the macrotask queue. This is why Promises execute before setTimeout even if setTimeout has 0 delay."
